# Contributing to Cairn

Cairn is an open-source project that belongs to everyone. Contributions of all kinds are welcome.

## Ways to contribute

### Code
- **Report bugs** — Open an issue with steps to reproduce.
- **Fix bugs** — Open a pull request with a clear description.
- **Add features** — Start a discussion first so we can align on design.

### Documentation
- Improve clarity, fix typos, add examples.
- Translate README or MANIFESTO into other languages.

### Community
- Share Cairn with someone who should record their family stories.
- Use it to archive your own family history and share what you learn.

### Ideas
- Open a discussion with your idea. The project is young and open to direction.

## Development setup

```bash
git clone https://github.com/cairn-project/cairn
cd cairn
npm install
npm test
```

## Guidelines

- Keep the code simple. Cairn prioritizes readability and reliability over cleverness.
- Follow the existing style (using this README as reference is fine).
- Write tests for new functionality.
- Data portability is a core value — never introduce a format that can't be easily exported.

## Philosophy

Before contributing, please read [MANIFESTO.md](./MANIFESTO.md). Cairn is guided by principles, not just features. Changes should align with:
- Data sovereignty (local-first, no cloud dependency)
- Long-term portability (plain formats)
- Warm, human-centered design
